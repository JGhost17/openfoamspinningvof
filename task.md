# OpenFOAM Mixing Tank — Task Reference

## Objective
Set up a two-phase (oil1/oil2) stirred-tank mixer mesh + simulation using OpenFOAM v2406, with a rotating impeller via AMI (Arbitrary Mesh Interface). Solve with **interFoam** (VOF) to extract transient velocity and phase fraction (alpha) fields until the two fluids are fully mixed (~50%/50% per cell).

## Geometry (STL files)

| File | Source | Description |
|---|---|---|
| `tank.STL` | CAD export (SLDPRT) | Outer tank walls — bounding box roughly (0,0,0)→(2.52, 2.52, 2.52) mm, scaled by 0.001 in snappyHexMesh |
| `impeller.STL` | CAD export | Spinning impeller, top sits 10 mm below tank top |
| `ami.STL` | Generated by `make_ami_closed.py` / `make_ami_open.py` | Cylindrical AMI interface centred at (1.27, 1.27), radius 0.4 m, spanning z [0.4, 2.315] |

**Python STL utilities:**
- `convert_stl.py` — Converts binary STL → ASCII STL (all 3 files)
- `make_ami_closed.py` — Generates a closed (watertight) cylindrical AMI surface
- `make_ami_open.py` — Generates an open (side-wall only) cylindrical AMI surface
- `read_stl.py` — Reads and prints STL facet count
- `read_stl_bbox.py` — Computes bounding box of an STL file

## Mesh Execution Order (strict)

1. `surfaceFeatureExtract` — extracts feature edges from tank.stl, impeller.stl, ami.stl
2. `blockMesh` — generates background hex mesh (68×68×63 cells)
3. `snappyHexMesh -overwrite` — snaps to STL surfaces, creates rotatingZone cellZone/faceZone
4. `renumberMesh -overwrite` — renumbers cells for optimal solver performance
5. `createPatch -overwrite` — splits rotatingZone faceZone → cyclicAMI patches ami1/ami2
6. `checkMesh` — validates mesh quality

---

## Simulation Execution Order

7. `setFields` — initialises oil1 phase fraction (bottom half: z < 1.17 m)
8. `interFoam` — transient VOF simulation with rotating AMI motion

## Key Parameters

### Mesh (coarse phase)
- Background cell size: ~1 cm (68×68×63 blocks in 2.72×2.72×2.52 m box)
- Refinement: impeller & AMI at level (2,2), tank at (0,0)
- No layers (addLayers=false) — deferred to refinement phase

### Motion
- Solver: `solidBody` via `dynamicMotionSolverFvMesh`
- Cell zone: `rotatingZone` (defined by snappyHexMesh via ami.stl)
- Rotation axis: (0 0 1) through origin (1.27 1.27 2.31)
- Angular velocity: 25 rad/s (~239 RPM)

### Fluids
- **oil1** (heavy phase, initially bottom): `nu=4.348e-5 m²/s`, `rho=1150 kg/m³`
- **oil2** (light phase, initially top): `nu=1.0e-5 m²/s`, `rho=1000 kg/m³`
- Surface tension: `sigma=0.07 N/m`

### Solver
- `interFoam` with VOF (MULES)
- PIMPLE: 3 outer correctors, 2 inner correctors, consistent=true
- Adjustable time-stepping: maxCo=5, maxAlphaCo=1, initial deltaT=0.001
- Turbulence: k-omega SST (k/omega fields at boundaries with wall functions)
- endTime=5 s (functionality check), writeInterval=0.1 s

### Boundary conditions
| Patch | U | p_rgh | alpha.oil1 | k | omega |
|---|---|---|---|---|---|
| tank | noSlip | fixedFluxPressure | zeroGradient | kqRWallFunction | omegaWallFunction |
| impeller | noSlip | fixedFluxPressure | zeroGradient | kqRWallFunction | omegaWallFunction |
| ami1 | cyclicAMI → ami2 | cyclicAMI → ami2 | cyclicAMI → ami2 | cyclicAMI → ami2 | cyclicAMI → ami2 |
| ami2 | cyclicAMI → ami1 | cyclicAMI → ami1 | cyclicAMI → ami1 | cyclicAMI → ami1 | cyclicAMI → ami1 |
